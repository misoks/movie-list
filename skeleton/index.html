Skeleton

Welcome to Skeleton, a framework for prototyping interactive UIs quickly and easily. Skeleton consists of a CSS file and a Javascript file that work together to take the hassle out of styling common UI elements and emulating common forms of interactivity. It is visually influenced by both Epic's style guide and Google's Material Design.

What can it do?

This guide covers most of Skeleton's current functionality and demonstrates how to use it. If you have any requests for future enhancements, contact Cathy Fisher.

How do I use it?

Use SVN to check out your own copy of Skeleton from this repository:

http://einstein/svn3/misc/UX/Skeleton

Use the provided template.html as a jumping-off point for building your own prototype. These styles are meant to be extended and tweaked as needed in your project, so feel free to override.

Happy prototyping! :)

Accessibility

Skeleton has two additional modes that allow for use in both dark environments and for users with impaired vision: Night Mode and High-Contrast Mode. Both will affect the colors used on most elements, including icons. High-Contrast Mode will also increase the sizes of fonts and various elements.

Add the night-mode or high-contrast class to the html element of your prototype to enable them.

Icons

Everyone loves SVGs because of their resolution-independence and stylability, but including SVG code in your HTML can be messy and annoying. Skeleton lets you easily swap in fully stylable SVG code with an image tag! Just insert the following code where you want your SVG to go, with the value of the filename attribute being the name of your SVG file (put it in the Icons folder):

<img class="vector" filename="folder"/>

Interactivity

Selectable Elements

Make an element in your prototype selectable by adding the .selectable class to it:

Select me!

Once you've selected something, you can use the up and down arrow keys to move through the selectable elements on the page.

  • I'm selectable
  • Me too
  • Me three

We also store the ID attribute of whatever element is currently selected in the currentSelected variable in case you want to use it to trigger some other behavior.

If you have an element you want to encourage the user to click on, but isn't selectable, add the .interactable class to it. It will have a hover effect and change the cursor to a pointer but can't be selected. This can be helpful to illustrate behavior and suggest click targets to users who are new to a UI.

Changing State

If you want your prototype to be able to move between various modes and states, you can do that using a .dynamic-area container.

Currently only slide-left, slide-right, slide-up, slide-down, and crossfade are available. The .mobile class is only specifying the .dynamic-area's size in this case. You can make the area any size or shape you want. If a transition attribute is not specified, there will be no animation.

As with forms, Skeleton will remember the state of a .dynamic-area container for the length of a session.

Utilities

Debugging

Skeleton has a built-in debugging area where you can dump values in a way that's a little less annoying or disruptive than alert(). Instead you can use debug(). It can handle various data types, including strings, numbers, and arrays (which it will print all the contents of).
